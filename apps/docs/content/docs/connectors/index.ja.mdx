# コネクタ

現在の `syncdown` には次のコネクタがあります。

- [Notion](/docs/connectors/notion)
- [Gmail](/docs/connectors/gmail)
- [Google Calendar](/docs/connectors/google-calendar)
- [Google Keep](/docs/connectors/google-keep)
- [Apple Notes](/docs/connectors/apple-notes)

いずれも同じレンダラーとファイルシステム sink を通して Markdown を書き込みますが、認証フロー、検出モデル、同期セマンティクスは異なります。

## 共通の挙動

すべてのコネクタについて:

- 同期前に認証情報を検証します。
- 同期開始前にバリデーションを実行します。
- コネクタ/integration 単位の Markdown パスに出力します。
- 後続の実行では既存の増分同期状態を再利用します。

## コネクタ別ガイド

- [Notion](/docs/connectors/notion) ではトークンと OAuth の設定、ページ検出、データソース動作を扱います。
- [Gmail](/docs/connectors/gmail) では Google OAuth、Primary 範囲の同期フィルタ、Gmail history ベースの増分挙動を扱います。
- [Google Calendar](/docs/connectors/google-calendar) では共有 Google OAuth、カレンダー選択、イベントの増分同期を扱います。
- [Google Keep](/docs/connectors/google-keep) ではマスタートークン認証、ノート/チェックリストの Markdown 出力、増分同期を扱います。
- [Apple Notes](/docs/connectors/apple-notes) では macOS 専用のローカルアクセス、アカウント/フォルダ出力パス、現在の制限事項を扱います。
